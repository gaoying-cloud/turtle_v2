# 海龟 V5.15 Dry-Run 每日操作手册

> 适用场景：dry-run 模拟盘阶段  
> 工具：`scripts/daily_signal.py`  
> 策略参数：ATR=15, Breakout=20, Stop=12, 1.5×ATR, α=0, Mode A（纯多头，4 品种）  
> 核心原则：**脚本出信号，人手执行；脚本不替你做决策，只告诉你该看什么**

---

## 一、每日节奏

```
16:30  py scripts/daily_signal.py           # 拉数据 → 算信号 → 出明日操作表
      ↓ 把操作表截图/打印/放桌面上

次日 9:30  按表手工下单                         # 记下实际成交价
次日 16:30 py scripts/daily_signal.py --settle 513100=22.38,518880=10.55
                                             # 结算今日成交 → 出明日操作表
```

> **为什么是 16:30 而不是 15:00？**  
> Tushare 日线数据通常在 16:00-17:00 才更新。15:00 跑会拿到昨天的数据。

> **如果 16:30 还没跑**：当天任何时间都可以跑，直到第二 9:30 前都来得及。

---

## 二、首次使用

```powershell
# 1. 确保数据完整（拉取 2014 至今所有数据）
py scripts/pull_data.py

# 2. 首次运行信号脚本（自动创建 data/daily_state.json）
py scripts/daily_signal.py

# 3. 查看输出确认一切正常
```

首次运行时 `data/daily_state.json` 自动生成，默认权益 ¥120,000、空仓。

> **如果让你输入实际账户权益**：打开 `data/daily_state.json`，修改 `"equity"` 的值为你实际的本金。

## 三、信号表解读

```
══════════════════════════════════════════════════════════════
  海龟 V5.15 每日信号   2026-06-22 (周一)
══════════════════════════════════════════════════════════════
  品种        持仓      操作    方向   数量     入场参考    止损
──────────────────────────────────────────────────────────────
  518880.SH   400股     持有     —      —      9.777    10.55
  513100.SH   空仓      买入    做多   400     22.350   21.80      ← 明天做这个
  510500.SH   空仓      暂停     —      —       —        连亏5次    ← 别碰
  159915.SZ   空仓       —       —      —       —        (无信号)
──────────────────────────────────────────────────────────────
  权益: ¥128,500 | 持仓品种: 1/4
  提醒: ⚠️ 518880 持仓已18天，别乱动
```

### 操作列含义

| 操作 | 含义 | 明天 9:30 做什么 |
|:--|:--|:--|
| **买入 / 做多** | 出现 20 日突破信号 | 按数量买入 |
| **持有** | 已持仓，未触发退出 | 不动 |
| **平仓** | 触发 10 日低点退出 | 全部卖出 |
| **加仓** | 持仓中，价格推进了 0.5N | 按数量加仓 |
| **暂停** | SignalFilter 连续拒绝 ≥3 次 | 不动，等解锁 |
| **— (无信号)** | 未达到突破条件 | 不动 |

### 止损说明

- **买入时**止损列显示的是 10 日低点，不是固定价格
- **持仓中**止损会随 10 日低点每天自动上移
- 脚本每个交易日重新计算，不用手动跟踪

## 四、手工下单 & 记录

建议用 Excel 或纸质记录每笔实际成交：

| 日期 | 品种 | 操作 | 脚本建议价 | 实际成交价 | 数量 | 滑点(bp) |
|:--|:--|:--|:--|:--|:--|:--|
| 6/23 | 513100 | 买入 | 22.35 | 22.38 | 400 | +0.13% |
| 6/23 | 518880 | 持有 | — | — | — | — |
| 6/24 | 513100 | 加仓 | — | 22.68 | 400 | — |

> 滑点 = abs(成交价 / 参考价 - 1) × 10000 (bp)。回测显示策略对 15bp 内滑点不敏感。

## 五、结算命令

下单后当天 16:30 跑结算：

```powershell
# 单品种结算
py scripts/daily_signal.py --settle 513100=22.38

# 多品种同时结算
py scripts/daily_signal.py --settle 513100=22.38,518880=10.55
```

结算后脚本自动：
1. 更新持仓的 `entry_price` 为你的实际成交价
2. 重新跑信号，输出次日的操作表

> **不要手改 `data/daily_state.json`**。结算请走 `--settle`。只有一种情况可以手改：首次运行时填入实际权益。

## 六、常见场景与处理

| 场景 | 你会看到 | 怎么做 |
|:--|:--|:--|
| 今天非交易日 | `[OK] 使用最近交易日数据` | 正常，信号仍有效 |
| 数据未更新 | `[WARN] 缓存最新日期=X` | 等 10 分钟重跑，或等 17:00 再试 |
| Tushare 挂了 | `[ERROR] 数据更新失败` | 用缓存数据出的信号，标个 ⚠️ |
| 持仓 >12 天 | 备注栏显示"别动" | **管住手** |
| 品种连亏 ≥5 次 | 备注栏显示"连亏N次" | 正常现象，按信号继续执行 |
| 信号显示"暂停" | 备注栏显示暂停原因 | 不动，等 SignalFilter 自动解锁 |
| 连续几周没信号 | 全部显示"无信号" | 正常，低频策略特征。2016年有过半年无信号 |

## 七、桌面上贴的四句话

这四条来自 102 笔交易、12 年回测的实证数据，实盘盯盘时看它们：

```
1. 连亏 5 次？正常。
   历史上最长就是 5 次，第 6 笔翻回来了。

2. 连亏是常态。
   45% 胜率 = 每 10 笔亏 5.5 笔。亏损是入场的门票。

3. 持仓 > 12 天？别动。
   持仓 1-10 天 → 0% 胜率（全部亏损）
   持仓 31-50 天 → 92% 胜率
   长持仓 = 趋势来了。管住手，等趋势自己走完。

4. 挨打要站直。亏损是成本，总要还回来。
   盈亏比 5:1 的前提是每一笔止损都执行到位。
   躲的结果往往是——小亏变大亏；或者连亏后不敢开仓，错过随后的大赢家。
```

## 八、已知限制

1. **脚本判断基于收盘价**，不处理盘中实时价格。如果盘中触发 10 日低点退出，脚本次日才知道。
2. **不加仓**（当前版本的 daily_signal 默认不做加仓信号）
3. **不做盘中风控**。超出 MAX_UNITS（4单位）或单个品种风险超额时脚本会退出，但不替你在盘中减仓。
4. **数据依赖 Tushare**。如果 Tushare 长时间不可用，信号出不来。
