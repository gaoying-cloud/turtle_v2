# 双引擎策略实验方案

> 生成日期：2026-06-26 | 当前版本：V5.17

---

## 一、问题背景

### 核心发现

纯海龟策略在全区间（2014-2026，12.4 年）表现：

| 区间 | CAGR | 夏普 | MDD | 胜率 | 盈亏比 |
|---|---|---|---|---|---|
| 全区间 2014-2026 | 16.10% | 0.37 | 16.2% | 44% | 4.27 |
| 样本外 2020-2026 | **9.64%** | 0.72 | 12.3% | 50% | 3.31 |

**策略正在退化**：“高收益高波动” → “低收益低波动”。风控始终在线（回撤稳定 <13%），但盈利能力在近 6 年明显衰减。

### 退化原因

- **2015 年 +128% 是异常值**——单年撑起了整个 12.4 年的回报
- **2020 年后 A 股结构突变**：外资+量化占比上升、市场有效性提高、假突破增多、板块轮动加速
- **海龟是“时间序列动量”策略**——吃绝对趋势。近 6 年 A 股没有全局性大牛市，而是“牛短熊长、宽幅震荡”，海龟频繁被两头打脸

### 核心假设

> 市场结构变了，策略必须进化。不是给海龟加过滤，而是根据市场状态切换策略：**趋势市用海龟（吃单边），震荡市/混沌市用其他策略（吃轮动/防守）**。

### 系统模块划分

```
              │ (已完工)   │  │ (待开发)   │
              └───────────┘  └───────────┘
```

### 模块一：状态机（Market Regime）

#### 三维度判定

使用 **252 日滚动分位** 替代绝对值阈值，消除不同年代的波动水平差异：

| 维度 | 趋势市 | 震荡市 | 混沌市 |
|---|---|---|---|
| ADX 分位 | `adx_rank > 0.75` | `adx_rank < 0.40` | `adx_rank > 0.90` |
| 价格位置 | `close > SMA60` (方向确认) | `close` 在 BB(20,2σ) 内 | — |
| 波动率 | `eff_20d > 0.60` (方向效率) | N 值分位 `< 0.30` | N 值分位 `> 0.85` |
| 额外保护 | — | — | 5 日回撤 `> 10%` (已有) |

#### 分位计算（关键：无未来函数）

```python
# ADX 的 252 日滚动分位（每个时点只看历史）
adx_rank = adx.rolling(252, min_periods=126).rank(pct=True)

# N 值分位同理
n_rank = atr.rolling(252, min_periods=126).rank(pct=True)
```

#### 状态切换策略

- **确认窗口**：连续 3 天同一状态才正式切换
- **过渡期（transitional）**：沿用上一策略，不做切换
- **混沌市**：两个引擎全部关闭，仅持有现金

### 模块二：引擎 A（海龟策略）

**状态：已完成（V5.17），原封不动。**

| 参数 | 值 |
|---|---|
| 入场 | 20 日高点突破 |
| 退出 | 10 日低点反向突破 |
| 加仓 | 每 0.5N 加一次，最多 4 单位 |
| 止损 | 2N ATR 跟踪 |
| 风控 | SignalFilter + 集中度熔断 + 连续亏损暂停 |

### 模块三：引擎 B（RSI 回归策略 — 待开发）

#### 设计原则

- **高胜率、低盈亏比**——与海龟相反，互补震荡市的特征
- **严格止盈**——触中轨即平仓，不恋战
- **紧凑止损**——0.5×ATR，因为震荡破位往往就是真突破

#### 入场规则

```python
# 信号条件
if RSI(14)[today] >= 30 and RSI(14)[yesterday] < 30:  # ← 穿越入场，不等首次跌破
    if close[today] > BB_lower(20, 2σ):               # ← 不在下轨下方
        if adx_rank < 0.40:                            # ← 确认震荡市
            signal = "buy"
```

**关键修正**：
- ❌ 原方案：`RSI < 30` 当天入场 → 暴跌中 RSI 会钝化在 20 以下数周
- ✅ 修正：RSI **从 <30 穿越回 ≥30** 当天入场 → 确认“止跌”信号

#### 信号上限保护

```python
# 防止恐慌日全品种同时触发 → 瞬间满仓抄底
if len(pending_signals) > 4:
    pending_signals = top_4_by_decline(pending_signals)
```

#### 退出规则

| 条件 | 优先级 |
|---|---|
| 止损：止损价被跌破 | 1（最高） |
| 止盈：RSI > 50 或 close > SMA20 | 2 |
| 自然退出：持仓 > 20 天无盈利 | 3 |

#### 仓位计算

```python
position_size = equity / N_positions / price    # 等额分配
stop_price = entry_price - 0.5 * ATR(20)        # 紧凑止损
```

#### Engine B 参数

| 参数 | 默认值 | 说明 |
|---|---|---|
| `rsi_period` | 14 | RSI 计算周期 |
| `rsi_entry` | 30 | RSI 穿越阈值 |
| `rsi_exit` | 50 | RSI 止盈阈值 |
| `bb_period` | 20 | 布林带周期 |
| `bb_std` | 2.0 | 布林带标准差倍数 |
| `stop_atr_mult` | 0.5 | 止损 ATR 倍数（海龟的一半） |
| `max_signals_per_day` | 4 | 每日最大信号数 |
| `max_holding_days` | 20 | 最大持仓天数 |
| `position_count` | 4 | 最大持仓数 |
| `exit_ma_period` | 20 | SMA 止盈周期 |

## 三、实验路线图

### S0：上界测试（30 分钟可完成）

#### 目的

用“上帝视角”回答一个根本问题：**“完美择时”下，双引擎理论上能不能跑赢纯海龟？**

#### 做法

```python
# 每天遍历
for t in range(1, len(data)):
    next_return = (close[t] - close[t-1]) / close[t-1]

    if next_return > 0:
        # 上帝知道涨 → 海龟满仓
        equity *= (1 + next_return)
    else:
        # 上帝知道跌 → 引擎 B 空仓/防守
        # 保守假设：引擎 B 在下跌日能赚半个日振幅
        equity *= (1 + 0.5 * abs(next_return))
```

#### 判定

| 结果 | 下一步 |
|---|---|
| 上界 > 纯海龟 | → **S1**，方向值得做 |
| 上界 ≤ 纯海龟 | → **放弃**，问题不在择时而在策略本身 |

```
纯海龟 (2020-2026): CAGR=9.64%  Sharpe=0.72  MDD=12.33%  Trades=44

要求引擎 B 至少满足一项：
  ✅ 夏普 > 0.72 (曲线更平滑)
  ✅ MDD < 12% (防守更好)
  ✅ 正收益且回撤可控 (能在震荡市存活)

❌ 如果一项都不满足 → 退回重新设计 Engine B
```

### S3：样本外冻结验证（2020-2026）

#### 做法

- 冻结 S2 调出的所有参数
- 用 **同一套硬编码切换表** 跑 2020-2026
- 对比纯海龟同期的 CAGR/夏普/MDD

#### 验收标准

```
双引擎 (2020-2026): CAGR=??%  Sharpe=??  MDD=??%

要求:
  ✅ CAGR > 9.6% (纯海龟同期)
  ✅ 或 Sharpe > 0.72
  ✅ 或 MDD 降低
```

> **为什么用 2014-2019 做样本内、2020-2026 做样本外？**
>
> 2015 牛市会把一切缺陷掩盖——用它调参就是过拟合牛市的特征。2020 年后 A 股已变（量化崛起、板块轮动），那个时代的参数如果还能跑赢纯海龟，才算抓住市场不变量。

### S4：动态状态机接入 + 摩擦成本容忍

#### 做法

- 用 **252 日滚动分位** 的动态状态机替代硬编码
- 在 2020-2026 上跑完整路由
- 衡量“切换摩擦成本”

#### 验收标准

```
动态状态机 vs 硬编码理想切换:

摩擦成本 = 硬编码_CAGR - 动态_CAGR

要求:
  摩擦 < 3% → 可接受 → 架构成功
  摩擦 3-5% → 需要优化状态机
  摩擦 > 5% → 状态机不可靠 → 退回硬编码定期再评估
```

#### 预期效果

```
CAGR 可能从 16% 降到 12%   (可接受)
同时 Sharpe 从 0.4 升到 0.8  (核心目标)
```
