# 自适应退出机制 — 待验证

## 来源
2026-07-08 讨论，基于 `tt.md` 提案的分析。

## 核心思路
非对称风控——趋势向上时宽容出场，趋势向下/盘整时严格出场。

## 提案原方案
- **状态识别**: `Close > MA60 且 MA60斜率>0` → 趋势向上
- **趋势向上时**: 用 Exit10（10日低点突破）——宽容，让利润奔跑
- **非趋势/向下时**: 用 MA20 止损——严格，快速离场

## 审查结论

### 不推荐原方案的原因
1. MA60 识别器太粗糙，滞后严重，且来回闪烁需要冷却期
2. 代码已有 `MarketRegime` 模块（基于 n_pct/eff_20d/n_trend），比 MA60 更科学
3. 提案对退出线位置关系的推论与代码实际使用的 Donchian 通道不完全一致
4. `stop_period=8` 已通过网格搜索 + OOS 验证，增量空间有限

### 2026-07-08 A/B 测试结果
现有的 `regime_filter`（MarketRegime 的 choppy 入口拦截）**显著劣化**：
- CAGR: 14.05% → 8.32%（-5.74pp）
- 夏普: 1.04 → 0.60
- 终值从 ¥517k 跌到 ¥271k

但这测试的是**入口拦截**（choppy 时不开仓），不是**出口切换**，所以并未否定"自适应退出"思路本身。

## 如果要验证，建议路径
1. 修改 `_should_exit`：趋势向上时 `stop_period=10`，非趋势时 `stop_period=6`
2. 状态识别用 `MarketRegime` 而非 MA60
3. 把"状态切换 stop_period"加入网格搜索参数空间进行验证
4. 在此之前应先调优 MarketRegime 的三个子指标权重和阈值
