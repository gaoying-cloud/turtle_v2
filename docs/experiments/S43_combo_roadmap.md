# 实验: 双策略组合路线图 (S43+)

## 元数据
- 提出: 2026-07-14
- 分支: `exp/S43_turtle_clean`
- 状态: ✅ Phase 1 完成 | 🔄 Phase 2a 相关性分析已完成（Pearson −0.02，组合评分 11/11）
- 前置: S42 (N字D点地板修复, 每日信号同步)

## 背景

N 字结构策略已到平台期：S42 OOS CAGR 6.1%, MDD 18.7%, 140 笔交易, S20 4/4 通过。大杠杆拉满，继续深挖边际收益递减且有过拟合风险。下一步转向：

1. **海龟策略重新验证** — 对标 N 字的 IS-only 扫参 → OOS 单次验证，消除参数泄露
2. **双策略组合** — N字+海龟，两个低相关策略的组合 alpha
3. **实盘一致性** — 确保 daily_signal 输出与回测一致

## 路线总览

```
Phase 1: 海龟干净重验证  ──→  IS-only 参扫 + OOS 单次验证 + 基线更新
         ↓                       产出：可信的海龟基准数字
Phase 2: 两策略组合优化   ──→  相关性分析 + 资金分配方案 + 组合回测
         ↓                       产出：N字+海龟组合策略
Phase 3: 实盘回测一致性    ──→  代码审查 + 历史回放验证
                                产出：可上线的实盘信号系统
```

## Phase 2: 两策略组合优化

### 实验: S44_strategy_combo

**前置**: Phase 1 完成 (海龟干净基线确立)。

**核心目标**: 构建组合回测框架，实现 L1 等权组合生产基线，探索 L2-L4 可选优化。

**预分析结论** (2026-07-14): 两策略日收益率相关性 −0.02（几乎为零），回撤重叠率仅 0.1%，50/50 等权即可将 MDD 从 18.5% 减半至 9.5%、Sharpe 从 1.25 提至 1.36。组合价值极高，L1 已是合格生产基线。

### 子任务

#### 2a. 双策略相关性分析 🔴 P0 — 核心部分已完成

**分析维度**:

1. ✅ **日收益率相关**: 两策略在 OOS 区间的日收益率斯皮尔曼相关系数
2. ✅ **最大回撤重叠**: 回撤区间是否重合（都亏钱的时期）
3. ⬜ **市场状态分解**: 按市场状态（趋势/震荡/高波/低波）分类，各自表现
   - 利用现有的 `market_regime.py` 做状态分类
4. ⬜ **交易信号重叠**: 同一时段两策略是否同时持仓/同时空仓

**已完成产出** (2026-07-14):
- 相关性矩阵 + 滚动相关性图 → `results/combo_analysis.json`
- 回撤重叠日历 → 两策略主回撤期完全不重叠（N字 2022, 海龟 2024-2025）
- 逐年收益对比 → 6/7 年双正, 0/7 年双负

**核心发现**:

| 指标 | 数值 | 判定 |
|:--|:--|:--|
| Pearson 日收益相关 | **−0.02** | 极低 — 几乎为零 |
| Spearman 秩相关 | **−0.05** | 极低 |
| 异向概率 | 97.0% | 两策略几乎不同向运行 |
| 滚动 12 月相关 < 0.3 | 100% | 所有时间窗口均低相关 |
| 回撤重叠率 (>3%) | 0.1% | 仅 1 天重叠 |
| 同时极端回撤 (>10%) | 0 天 | 完全无重叠 |
| 双正/双负年份 | 6 / 0 (7 年) | 无一年两策略同时亏损 |

**L1 等权组合预分析** (50/50 简单日频再平衡):

| 指标 | N字 S42 | 海龟 S43 | **50/50 组合** | 改善 |
|:--|:--:|:--:|:--:|:--|
| CAGR | 8.0% | 18.8% | **13.9%** | — |
| 年化波动 | 14.1% | 15.1% | **10.2%** | **−30%** |
| Sharpe | 0.57 | 1.25 | **1.36** | 超最优 +9% |
| MDD | −17.8% | −18.5% | **−9.5%** | **减半** |
| Calmar | 0.45 | 1.02 | **1.45** | +42% |

**决策点** (已明确): 相关性 −0.02 ≪ 0.3，组合价值极高。L1 等权已全面超越所有 Phase 2 成功标准（Sharpe 1.36 > 1.25，MDD −9.5% > −18.5%）。L2-L4 的优化目标是：在 L1 基础上进一步改善 Sharpe/MDD，但 L1 本身已经是合格的生产基线。

#### 2b. 资金分配方案设计 🔴 P0

**由简到繁，逐级测试**:

| 级别 | 方案 | 说明 | 状态 |
|:--|:--|:--|:--:|
| L1 | 等权 50/50 | 最简单，无参数 | ✅ 预分析完成：Sharpe 1.36, MDD −9.5% |
| L2 | 波动率倒数加权 | 低波策略多配，高波策略少配 | ⬜ 可选 — 仅当能超越 L1 时采纳 |
| L3 | 风险平价 | Ledoit-Wolf 协方差估计（复用 `risk_parity.py`） | ⬜ 可选 — 两策略相关 −0.02，协方差 ≈ 0，增量有限 |
| L4 | 市场状态切换 | 趋势市偏海龟，震荡市偏 N 字 | ⬜ 可选 — 最有潜力的优化方向 |

> **L1 定位变更**: 原路线图将 L1 视为"起点"，但预分析表明 L1 已全面超越所有 Phase 2 成功标准。
> L1 现在是**生产基线**，L2-L4 是**可选增强**。2c 组合引擎应优先实现 L1 并保证其正确性，
> L2-L4 作为可插拔分配策略在引擎中留好扩展点。

**设计原则**:
- 共享资金池 (100K)，独立交易逻辑
- 总敞口上限 (如 150%)
- 单策略最低分配保障 (如 30%)，防止一方被饿死
- 再平衡周期 (周/月/季度)

#### 2c. 组合回测框架 🔴 P0

**新建**: 不修 `run_portfolio`，从零构建干净的组合层。

```python
# 新文件: strategies/combo_engine.py
class ComboEngine:
    """双策略组合引擎。

    设计:
      - 两个策略独立运行各自的信号逻辑 (不修改 n_structure / turtle_trading)
      - 组合层只负责: 资金分配 + 敞口检查 + 净值汇总
      - 日频循环: 信号扫描 → 资金分配 → 执行 → 净值更新
    """
```

**关键设计**:
- 策略信号接口：统一两个策略的输出格式（进场/加仓/退出）
- 资金竞争规则：同日多信号按质量排序分配
- 组合净值 = 现金 + N字持仓市值 + 海龟持仓市值
- 日频 diagnostics: 每个策略的资金使用率、贡献度

#### 2d. 组合回测与对比 🔴 P1

```bash
py scripts/run_combo.py --start 2020-01-01  # OOS 验证
```

**产出**:
- 组合净值 vs N字单独 vs 海龟单独 (叠加图)
- 逐年收益对比表
- 滚动 12 月 Sharpe/MDD 对比
- 归因分析：哪个策略贡献了最多 alpha

### 成功标准

| 指标 | L1 等权实际 | L1 门槛 | L2-L4 优化目标 | 说明 |
|:--|:--:|:--:|:--:|:--|
| 组合夏普 | **1.36** | > max(海龟, N字) = 1.25 ✅ | > 1.36 | L2-L4 需超越 L1 才有意义 |
| 组合 MDD | **−9.5%** | < 更好(海龟, N字) = −17.8% ✅ | < 9.5% | L1 已减半，L2-L4 需更低 |
| 组合 CAGR | **13.9%** | 不要求超过 18.8% | ≥ 13.9% | L1 已有竞争力 |
| 策略存活 | — | 两个都分到资金 | 最低 30% 保障 | — |

> **L1 等权已全面达标**。L2-L4 的优化价值在于：能否在不牺牲 CAGR 的前提下进一步降低 MDD，或在同等 MDD 下提升 CAGR。即使 L2-L4 无法超越 L1，L1 本身已经是合格的生产基线。

## Phase 3: 实盘回测一致性

### 实验: S45_live_consistency

**前置**: Phase 1+2 完成，策略逻辑定型。

**核心目标**: 确保每日实盘信号与回测输出完全一致。

### 子任务

#### 3a. 海龟 daily_signal.py 代码审查 🔴 P0

对标 N 字的 GLM 交叉审查，对海龟实盘脚本做同样审查：

- 与 `turtle_trading.py` 的逻辑一致性对比
- 未来函数检查（实盘中能否复现回测的信号）
- 状态持久化正确性（state.json 的跨日一致性）

**审查维度**:
1. 入场条件对齐: 唐奇安通道判断 / MA10 金叉 / ATR 计算时机
2. 出场条件对齐: 移动止损 / 加仓止损更新 / N值退化检测
3. 仓位计算对齐: ATR 仓位 × 集中度衰减 × 风险平价
4. 熔断/风控对齐

#### 3b. N 字结构历史回放验证 🔴 P1

**方法**: 用同一段历史数据，对比 daily_signal 日频输出 vs 回测 Trade 列表。

```bash
# 新脚本或改造 daily_signal_n_structure.py
py scripts/verify_daily_signal_ns.py --start 2024-01-01 --end 2026-07-10
```

**验证点**:
- 每个交易日: daily_signal 输出信号 vs 回测是否同日触发
- 每笔交易: 进场价/出场价/出场原因/盈亏 是否一致 (允许 ±0.5% 误差)
- 状态一致: 持仓/权益/连续亏损熔断
