## 子任务 0.1: 修复 N字结构识别中的未来信息泄露 🔴

### 问题根因

`strategies/n_structure.py` 的 `find_n_structure_in_window()`:

```python
a_local_idx = window['low'].idxmin()     # 看完整窗口 → 全局最低点
d_local_idx = after_a['high'].idxmax()   # 看 A 之后全部数据
b_local_idx = after_d['low'].idxmin()    # 看 D 之后全部数据
```

三个点都依赖窗口内的全局 min/max 操作，实盘中只有窗口"走完"后才能确认。

### 修复方案: 局部极值 + 确认延迟

```
新逻辑:

1. A点 — 局部低点 + 确认延迟:
   - 低点左右各 left_k/right_k 根 K 线价格不低于它
   - 低点出现后至少 confirm_k 根 K 线，未再创新低
   → 此时 A 可确认

2. D点 — A 之后的局部高点:
   - 高点左右各 left_k/right_k K 线价格不高于它
   - D > A × (1 + min_advance_pct)  # 有意义反弹 ≥ 5%
   - A→D 中间至少 min_gap 根 K 线 (≥5根)
   → D 可确认（但 A-D 段至此全暴露）

3. B点 — D 之后的局部低点:
   - 低点左右各 left_k/right_k K 线不低
   - A < B < D  (更高低点)
   - D→B 中间至少 min_gap 根 K 线 (≥3根)
   - B 出现后至少 confirm_k 根 K 线未跌破
   → B 可确认

4. 结构完成后，在 B 确认的下一根 K 线触发进场检查
```

### 新增参数

```python
# 局部极值参数（可配置）
local_left: int = 3       # 局部极值左侧比较窗口
local_right: int = 3      # 局部极值右侧比较窗口
confirm_k: int = 2        # 极值确认延迟（多少根 K 线不刷新才确认）
min_advance_pct: float = 0.05  # D > A 的最小幅度
min_gap_ad: int = 5       # A→D 最小间距
min_gap_db: int = 3       # D→B 最小间距
```

### 改动文件
- `strategies/n_structure.py` — `find_n_structure_in_window()` 重写
- 新增 `_find_local_min()` / `_find_local_max()` 辅助函数

## 子任务 0.2: 日频净值曲线

### 问题

`compute_metrics()` 的 Sharpe/MDD 基于交易序列 PnL，非日频：
```python
equity = initial_capital + np.cumsum(pnls)  # 逐笔，非逐日
```

### 修复方案

1. `NStructureStrategy.run()` 返回每日权益 Series:
   ```python
   # equity[i] = cash + sum(positions × close[i])
   # 返回值: (df_result, trades, equity_curve)
   ```

2. `compute_metrics()` 新增 `daily_equity` 可选参数:
   ```python
   def compute_metrics(trades, ..., daily_equity=None):
       if daily_equity is not None:
           # 用日频收益率计算 Sharpe
           daily_returns = daily_equity.pct_change().dropna()
           sharpe = daily_returns.mean() / daily_returns.std() * sqrt(252)
           # 用日频权益计算 MDD
           peak = daily_equity.expanding().max()
           mdd = ((peak - daily_equity) / peak).max()
   ```

3. `scripts/run_n_structure.py` — 适配新返回值

## 子任务 0.3: 摩擦成本模型

### 修复方案

`NStructureStrategy.__init__` 新增参数:
```python
slippage_pct: float = 0.001     # 每笔成交滑点
commission_pct: float = 0.00015 # 费率
```

所有成交价应用摩擦:
```python
# 买入价 = open × (1 + slippage_pct)
# 卖出价 = exit_price × (1 - slippage_pct)  
# 每笔交易额外扣: 成交金额 × commission_pct
```

CLI 新增参数:
```bash
--slippage 0.001 --commission 0.00015
```

### 结论

✅ **通过** — 诚实基线显示 N字策略在 12.5 年全周期上：
- **CAGR 17.7% 远超 10% 底线**，仅比旧基线低 0.8pp
- MDD 21.3% 远高于旧基线 8.6%，但这才是**真实的回撤水平**
- 608 笔交易（月均 4 笔），6 品种全部盈利
- 策略表现出典型趋势跟踪特征：少数大赢家（跟踪止损）承担全部利润，多数小亏损（B点失效/D点失败）做过滤成本

**关键洞察**：
1. 信息泄露修复的影响远小于预期（-0.8pp CAGR），说明旧版的主要利润来源不是"偷价"，而是真正的趋势跟踪
2. 日频 MDD 是交易序列 MDD 的 2.5 倍 — 这个差异必须在实盘中充分预期
3. B点结构失效（181 笔, 4.4% 胜率）是最值得优化的退出原因 — 考虑放宽 B 点跌破阈值
4. D点突破失败（228 笔, 63% 胜率, 微利）是无效持仓 — 考虑将 5K 超时延长或直接移除这个退出条件
