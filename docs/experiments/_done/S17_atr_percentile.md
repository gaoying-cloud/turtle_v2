## 实验设计

### 参数网格
```bash
thresholds = [0.0, 0.5, 0.6, 0.65, 0.7, 0.75, 0.8, 0.85, 0.9]
```
- `0.0` = 关闭过滤（等价于基线）
- `0.5` = 过滤掉 ATR 处于历史前 50% 的入场信号（最宽松过滤）
- `0.9` = 仅过滤掉 ATR 处于历史前 10% 的极端高波动入场（最严格）

### 运行
```bash
for t in 0.0 0.5 0.6 0.65 0.7 0.75 0.8 0.85 0.9; do
  echo "=== threshold=$t ==="
  py scripts/run_backtest.py --atr-pct-filter $t 2>&1 | tail -12
done
```

**关键发现**：阈值在 0.75 附近形成一个清晰的峰值，过紧（0.5-0.6）或过松（0.85-0.9）效果都下降。这说明 ATR 百分位过滤的有效性集中在**过滤前 25% 的最高波动期**——只砍掉极端高波动的假突破，而不是大面积过滤。

### 结论

✅ **实验通过。** 推荐将 `atr_pct_threshold = 0.75` 定型为基线配置。
