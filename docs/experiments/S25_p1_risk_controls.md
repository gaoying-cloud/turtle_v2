## 子任务 1.1: 动态权益仓位计算

### 方案

用当前权益（而非固定 `capital_per_symbol`）计算每笔仓位：
```python
equity_for_sizing = current_equity if self.use_dynamic_equity else self.capital_per_symbol
shares_per_unit = self._calc_shares(equity_for_sizing, entry_price, atr)
```

`run()` 中维护 `current_equity`（已实现权益），每次平仓后 `current_equity += trade.pnl`。
权益曲线 = `current_equity + unrealized_pnl`。

### 状态: ✅ 完成

---

## 子任务 1.2: 连续亏损熔断

### 方案

新增参数 `max_consecutive_losses=5`, `pause_bars=20`。
每次亏损平仓后 `consecutive_losses += 1`，盈利后归零。
达到阈值后暂停 `pause_bars` 根 K 线，跳过所有进场。

### 状态: ✅ 完成

验证（阈值=2 强制触发）：交易从 98 → 72 笔（-26%），CAGR 不变（16.7% vs 16.8%），机制正常。
默认阈值=5（42% 胜率下概率 ~6.6%），自然触发频率合理。

---

## 子任务 1.3: 加仓重算单位风险

### 状态: ⏸️ 推迟到 Phase 2

原因：加权平均入场价的简单实现会破坏 PnL 计算。
正确实现需要按单位独立追踪（每单位有自己的 entry_price 和 shares），涉及仓位数据结构重构，放到 Phase 2 组合回测时一并做。
