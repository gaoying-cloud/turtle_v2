### 新发现：退出逻辑有根本性缺失

对比两个系统的退出检查：

**经典海龟交易法则的 A 股 ETF 趋势跟踪系统 `_check_exits()`**（完整）：
```
① 固定止损（close ≤ entry - 2N）
② 移动止损（close ≤ trail_high_10 - 2N，只上移）  
③ 10日反向突破（low ≤ stop_low_10）
→ 三者取最早触发者
```

**turtle_v2 `_should_exit()`**（残缺）：
```
仅 10日反向突破（low ≤ stop_low_10）
```

**turtle_v2 的 `_should_exit()` 完全不检查 `pos.stop_loss`！** 入场时虽然计算了止损线，但逐日 `next()` 中从未读取它——只在 docstring 中描述了"固定止损"和"移动止损"，代码里都没实现。

### 结论：差异原因已经变化

| 原因 | 此前判断 | 现在判断 |
|------|:---:|:---:|
| P0-1 累计亏损永停 | 主因 | ✅ 不存在 |
| P0-2 移动止损冻结 | 主因 | ❌ 不止冻结，是**完全缺失** |
| 2N 固定止损不执行 | 未发现 | 🔴 **新发现的主因** |
| 移动止损完全不更新 | 次要 | 🔴 **复合问题** |

**核心结论**：turtle_v2 的退出逻辑等于从完整的三重保护（2N止损+移动止损+10日突破）退化为只有 10 日突破单一退出。对于宽幅震荡市场，10 日低点可能比 2N 止损线低很多，会导致亏损放大；对于强趋势，缺少移动止损上移保护，过早被震出。这是 turtle_v2 绩效差于主系统的**最根本原因**——不是"策略设计不同"，而是退出逻辑有重大遗漏。

### 需要改 3 处

#### 1. `src/turtle_core.py` — 新增 `should_activate_trailing_stop()` 函数

```python
def should_activate_trailing_stop(
    current_price: float, entry_price: float, n_value: float,
    holding_days: int = 0, profit_threshold_n: float = 2.0, days_threshold: int = 20,
) -> bool:
    if n_value <= 0:
        return False
    floating_profit_n = (current_price - entry_price) / n_value
    return floating_profit_n >= profit_threshold_n or holding_days >= days_threshold
```

#### 2. `strategies/turtle_trading.py` — 修改 `_should_exit()` 增加 2N/移动止损检查

在现有的 10 日突破检查之前，增加：
- 固定止损：`close <= pos.stop_loss`
- 移动止损：同上（止损线由 `_update_trailing_stop` 每日上移）

#### 3. `strategies/turtle_trading.py` — 新增 `_update_trailing_stop()` 并在 `next()` 中每日调用

- 浮盈 ≥ 2N 或持仓 ≥ 20 天后激活移动止损
- 每日更新 `trail_high_10` → 计算新止损线（只上移）
- `next()` 中对每个持仓品种先调用 `_update_trailing_stop`，再检查退出

同时补充 import：`calc_trailing_stop`、`should_activate_trailing_stop`
