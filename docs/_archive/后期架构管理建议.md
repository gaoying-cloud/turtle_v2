**按优先级排序：**

### 值得做 ✅

| 优先级 | 事项 | 理由 |
|:--:|:--|:--|
| **P0** | **SQLite / DuckDB** | 替代 Parquet 文件散落，统一管理行情+回测结果，支持 SQL 查询，DuckDB 零配置且对 pandas 友好 |
| **P1** | **FastAPI + 简单前端** | 把回测/网格搜索/报告做成 Web 界面，比命令行方便太多。轻量 SPA（Vue/React）或直接用 Gradio/Streamlit |

### 不值得做 ❌

| 事项 | 理由 |
|:--|:--|
| **ORM（SQLAlchemy 等）** | 量化项目数据结构简单（OHLCV + 信号），手写 SQL 或 pandas 直接读写更高效，ORM 是过度设计 |
| **前端状态管理（Redux/Pinia）** | 如果只是展示回测结果，组件内状态 + API 请求足够，不需要全局状态管理 |
| **API 路由规范（RESTful 严格分层）** | 内部工具项目，FastAPI 几个端点就够，不需要严格的资源路由体系 |

### 建议路线

```
当前 CLI 脚本 → S9 Dry-Run 验证 → 加 DuckDB 统一存储 → 加 Streamlit/Gradio 轻量 UI
```

**Streamlit 最推荐**：纯 Python，零前端代码，几小时就能把回测结果做成交互式 Dashboard，不需要 API 路由、不需要前端框架、不需要状态管理。
