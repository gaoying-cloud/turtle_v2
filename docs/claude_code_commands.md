## 📌 高频速查 — 日常场景对应命令

| 场景 | 用这个命令 | 一句话说明 |
|------|-----------|-----------|
| 🐛 **改完代码检查有没有 bug** | `/code-review medium` | 审查当前 diff 的正确性问题 |
| 🧹 **改完代码想顺便优化质量** | `/simplify` | 自动应用简化/复用/效率改进 |
| ✅ **改动较大，确认没改坏** | `/verify` | 端到端跑一遍实际流程验证 |
| 🧪 **跑测试** | `! py -m pytest tests/ -q` | `!` 前缀直接执行本地命令 |
| 📋 **有多个任务要追踪** | `/tasks` | 创建、查看、标记任务进度 |
| 🔄 **定时巡检/重复操作** | `/loop 10m /verify` | 每 10 分钟自动验证一次 |
| 🎯 **复杂功能不知从哪下手** | `/plan` | 进入计划模式，先设计后实施 |
| 🔍 **研究一个陌生主题** | `/research <topic>` | 多路搜索+溯源+生成引用报告 |
| 🚀 **启动当前项目** | `/run` | 自动检测项目类型并启动 |
| 🔐 **代码安全意识强** | `/security-review` | 审查分支变更的安全风险 |
| ⚡ **嫌回复太慢** | `/fast` | 切换快速模式，输出更快 |
| 🧠 **换个更强的模型做分析** | `/model claude-opus-4-8` | 切换到 Opus 做复杂推理 |
| 🗑️ **会话太长有点卡** | `/clear` | 清空上下文释放空间 |
| 📝 **初始化/更新项目文档** | `/init` | 生成或更新 `CLAUDE.md` |
| 🔧 **想改 Claude Code 设置** | `/config` | 修改全局/项目配置 |
| 🔇 **权限弹窗太多** | `/fewer-permission-prompts` | 把常用命令加入白名单 |

## 一、会话管理

| 命令 | 说明 |
|------|------|
| `/clear` | 清除当前会话上下文（释放上下文窗口，长会话后使用） |
| `/model [name]` | 切换 AI 模型。不传参查看当前模型，传参切换并保存为默认 |
| `/fast` | 切换快速模式开关（Opus 更快输出，不降级模型） |

### 会话生命周期

| 机制 | 说明 |
|------|------|
| **上下文摘要** | 长会话达到上下文上限时，系统自动摘要早期内容，继续工作无需手动操作 |
| **记忆系统** | 持久化记忆保存在 `~/.claude/projects/<项目名>/memory/`，跨会话保留重要事实。使用 `/remember` 手动保存 |
| **`!` 前缀** | 在输入框中用 `! <command>` 执行本地 shell 命令，输出直接进入会话（如 `! gcloud auth login`） |
| **Cron 定时任务** | 通过 `/loop` 设定周期性任务（见下文），支持持久化（跨会话存活）和一次性提醒 |
| **后台 Agent** | 长时间运行的任务在后台执行，完成后自动通知，不影响当前对话 |

### 工作树隔离（Worktree）

| 命令 | 说明 |
|------|------|
| `EnterWorktree` | 创建隔离的 git worktree，适合并行开发互不干扰 |
| `ExitWorktree` | 退出 worktree（可选择保留或删除） |

## 二、开发工作流

| 命令 | 说明 |
|------|------|
| `/plan` | 进入计划模式 — 分析代码库、设计方案，经你批准后实施 |
| `/code-review [level]` | 审查当前 Git diff 的 Bug/正确性问题。level: `low`/`medium`/`high`/`xhigh`/`max` |
| `/fix` | 自动修复代码问题 |
| `/verify` | 端到端验证改动是否按预期工作（驱动实际流程而非仅跑测试） |
| `/simplify` | 审查并应用代码简化/复用/效率优化（不查 Bug） |
| `/security-review` | 对当前分支变更进行安全审查 |

## 三、任务与循环

| 命令 | 说明 |
|------|------|
| `/tasks` | 查看和管理任务列表（Todo List），支持依赖关系和进度追踪 |
| `/workflows` | 查看多 Agent 工作流的实时进度 |
| `/loop <interval> <command>` | 按间隔重复执行命令。`/loop 10m /code-review` 每 10 分钟审查一次 |
| `ScheduleWakeup` | 动态循环模式 — 自 pacing 任务迭代，智能选择唤醒间隔（60s–3600s） |

### Cron 定时任务细节

| 特性 | 说明 |
|------|------|
| **语法** | 标准 5 字段 cron（分 时 日 月 周），本地时区 |
| **一次性任务** | 设置 `recurring: false`，触发后自动删除 |
| **持久化任务** | 存到 `.claude/scheduled_tasks.json`，跨会话存活 |
| **自动过期** | 周期性任务 7 天后自动过期 |

## 九、命令使用示例

### 日常开发流程

```bash
# 1. 改完代码 → 跑测试
! py -m pytest tests/ -q

# 2. 代码审查（常规强度）
/code-review medium

# 3. 审查完自动修复
/fix

# 4. 端到端验证
/verify

# 5. 提交前检查安全
/security-review
```

### 复杂任务场景

```bash
# 不确定怎么实现 → 先计划
/plan

# 研究新技术/策略
/research "跨市场ETF组合的风险平价策略比较"

# 需要更强推理能力
/model claude-opus-4-8

# 启动应用看看效果
/run
```

### 会话杂项

```bash
# 切换模型（立即生效并保存为默认）
/model claude-sonnet-5

# 查看当前模型
/model

# 清空上下文（长会话后释放空间）
/clear

# 快速模式开关
/fast

# 每 5 分钟验证一次
/loop 5m /verify

# 初始化/更新项目文档
/init

# 在输入中执行本地命令（输出直达会话）
! git status
! py -m pytest tests/ -q
```

## 十、技巧与提示

1. **会话管理**：
   - 长会话会自动摘要，无需手动 `/clear`，除非想彻底释放上下文
   - 使用 `/remember` 保存重要信息到跨会话记忆
   - `!` 前缀执行本地命令，适合交互式登录（如 `! gcloud auth login`）

2. **命令链**：可在 `/loop` 中嵌套其他命令，实现自动化巡检

3. **权限白名单**：使用 `/fewer-permission-prompts` 可大幅减少权限确认弹窗

4. **模型选择**：
   - `claude-opus-4-8` — 最强推理，适合复杂分析
   - `claude-sonnet-5` — 速度与质量的最佳平衡
   - `deepseek-v4-flash` — 快速响应，适合简单任务
   - `claude-haiku-4-5-20251001` — 轻量快速，适合简单分类/提取

5. **/code-review 努力级别**：
   - `low/medium` — 高置信度发现，误报少
   - `high/max` — 更全面覆盖，可能包含不确定发现

6. **/plan 工作流**：进入计划模式 → 探索代码 → 设计方案 → 提交审批 → 自动实施

7. **后台任务**：使用 Agent 工具启动长时间运行的任务，完成后会自动通知你，不阻塞当前对话
