# T+0/T+1 结算规则差异对海龟策略的影响分析

## 1. 背景

本策略组合包含 A 股 ETF、跨境 QDII ETF 和商品 ETF，三者适用不同的结算规则。中国 A 股 ETF（含科创板ETF）为 **T+1 结算**（当日买入不可卖出），而跨境 QDII ETF、商品 ETF、债券 ETF 为 **T+0 结算**（当日可双向交易）。

结算规则差异直接影响海龟策略的**止损执行**和**加仓可逆性**，在回测中如果不加区分地统一假设"当日可买卖"，将导致绩效高估。

## 2. 组合结算规则分布

| 品种 | 代码 | 类别 | 结算规则 | 策略角色 |
|:--|:--|:--|:--:|:--|
| 中证500ETF | 510500.SH | A股核心宽基 | T+1 | 核心持仓 |
| 中证1000ETF | 159845.SZ | A股小盘 | T+1 | 核心持仓 |
| 创业板ETF | 159915.SZ | A股成长 | T+1 | 核心持仓 |

> **核心持仓中 4/6 为 T+1**，T+1 品种占策略核心仓位的大头。

## 3. 各场景影响分析

### 3.1 🔴 关键风险：T+1 品种止损滞后

**问题描述**：

海龟策略止损规则：价格反向突破过去 10 日低点（多单）或亏损达到 2N，取更早触发者。

```
Day T 09:35 — 触发 20 日突破买入，以价格 P 成交
Day T 10:15 — 价格急跌至 P - 2N，已触及 2N 止损线
        ❌  T+1 规则下当天无法卖出
Day T+1 09:30 — 开盘可能进一步跳空低开
        → 实际止损价  <<  理论止损价
```

**量化估计**：
- 假设日波动率 ~2%，T+1 止损滞后额外增加 **0.5%~1.5%** 的滑点损失
- 在 2020 年 3 月、2022 年 3-4 月等剧烈波动期，年化收益率被**高估 1~3%**，最大回撤被**低估 2~5%**

**适用品种**：中证500ETF、中证1000ETF、创业板ETF、科创50ETF

### 3.2 🟡 中等影响：日内加仓的流动性不对称

海龟加仓规则：每上涨 0.5N 加仓一次，最多 4 个单位。

| 场景 | T+1 品种 | T+0 品种 |
|:--|:--|:--|
| 日内价格一路上涨 +2N，触发 4 次加仓 | 买入信号密集触发但当日**无法卖出**已有仓位，加仓单向增加多头 | 4 单位加完后若价格急跌，当日**可平掉**部分仓位 |
| 加仓后价格反转 | 当日无法减仓，需承受隔夜风险 | 当日可灵活调整 |

**实际影响评级**：中低。理由：加仓策略本身基于趋势延续假设，趋势在 1 个交易日内反转的概率较低。回测中可暂不做修正，但须在 Dry-Run 阶段实测。

### 3.3 🟢 无阻塞：现金流切换

空仓期 → 入场切换过程：
- 卖出国债ETF（T+0，资金**实时可用**）→ 买入中证500ETF（当日买入，T+1 交割）

买入操作不依赖卖出结算完成，现金流无阻塞。

**适用场景**：所有海龟品种与国债ETF之间的资金切换。✅

### 3.4 🟡 需关注：纳指ETF T+0 + QDII 溢价叠加

纳指ETF 的 T+0 交易的是中国二级市场的人民币份额，价格 = MAX(NAV溢价/折价, 市场情绪)。T+0 意味着日内可反复买卖溢价本身。

海龟策略仅跟踪收盘价时，实际成交价与策略信号价可能偏差较大。

**应对措施**：
- 严格执行"溢价>3%暂不开仓"风控规则
- 在 Dry-Run 阶段监控溢价率波动

## 4. 回测修正方案

### 4.1 必须修正（高优先级）

在 `strategies/turtle_trading.py` 中增加 T+1 约束逻辑：

```python
# 伪代码 — 回测中 T+1 品种的止损修正
def handle_stop_loss(strategy, data, dt):
    """T+1 品种的特殊止损处理"""
    if data._name in T_PLUS_ONE_SYMBOLS:
        # 获取当日买入记录
        today_buys = get_today_buys(data._name, dt)
        if today_buys > 0 and stop_loss_triggered_same_day(data):
            # 当日买入后无法在当日止损，推迟至下一交易日开盘
            schedule_stop_loss_next_day_open(data._name)
            strategy.log(f"[T+1修正] {data._name} 当日买入+止损同一天触发，推迟至明日开盘执行")
            return  # 跳过当日止损
    # 正常止损逻辑
    execute_stop_loss(strategy, data)
```

### 4.2 建议修正（中等优先级）

```python
# 伪代码 — 日内加仓不可逆
def check_pyramid_add(strategy, data, dt):
    """T+1 品种的加仓不可逆约束"""
    if data._name in T_PLUS_ONE_SYMBOLS:
        if pyramid_add_triggered(data):
            # 记录加仓，但当日不可反向平仓
            record_one_way_pyramid(data)
    # T+0 品种可正常执行
```

### 4.3 不需要修正（低优先级）

- T+0 品种（黄金ETF/纳指ETF/国债ETF）的当日完整交易循环：已是保守假设，无需额外修正。

## 5. Dry-Run 实测要点

| 验证项 | 方法 | 通过标准 |
|:--|:--|:--|
| T+1 品种止损滑点 | 记录每笔止损交易的实际滑点，按品种分类统计 | 平均滑点 ≤ 0.5% |
| T+1 品种同日买卖冲突频率 | 统计买入信号与止损信号同一天触发的次数 | 占比 < 5%（正常情况下） |
| 纳指ETF溢价对信号的影响 | 对比收盘价信号 vs 实际成交价差异 | 溢价>3%时交易次数不高于正常水平 |

## 6. 长期关注

1. **A 股 T+0 ETF 试点**：科创板 ETF T+0 试点已列入讨论。如果未来有 A 股 T+0 ETF 上市，可考虑替换部分 T+1 品种，提升执行一致性。
2. **替代品种筛选**：若 T+1 止损滞后造成的实际损失超出预期，需考虑用相近波动特征的 T+0 品种替代（如将部分 A 股仓位转向纳指ETF同类或商品ETF）。
3. **盘中数据接入**：如果长期运行证明 T+1 滞后问题严重，需要考虑接入分钟级数据，在更细粒度上优化止损执行。
