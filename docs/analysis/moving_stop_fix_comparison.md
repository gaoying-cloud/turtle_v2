### 1. 方案A全面碾压

### 2. 为什么方案A有效？

**核心变化：退出规则从两条叠加重叠变为一条**

```
V1/V2 双重退出:
  close ≤ 2N追尾止损  ← 先触发，紧贴价格
  low ≤ 10日低点       ← 后触发，更宽松

V3 方案A 单退出:
  low ≤ 10日低点       ← 唯一退出条件
```

V3 删除了 `close ≤ pos.stop_loss` 这条退出线后：
- **盈利能跑**：不再因2N追尾止损线的紧贴被短期回调踢出（平均盈利 ¥2,117 → 几乎不变）
- **亏损可控**：10日低点比2N追尾更稳定（平均亏损从 ¥2,060 降至 ¥1,017，↓38%）
- **回撤缩小**：减少因过早止盈/过晚止损造成的效率损失

### 3. 经典海龟的正确性验证

方案A对应的是经典 Turtle 系统的退出规则：**跌破10日低点出场**。这个规则已经被市场验证40年以上。当前项目中 `trail_high_10` 实现的「最高收盘价 - 2N」追尾止损实际上是一个**对经典规则的自定义偏离**，而这个偏离在 A 股 ETF 这种趋势持续性较弱的市场中产生了负面影响。

### 4. 后续关注点

- **参数调优空间**：虽然方案A已大幅改善，但参数仍可进一步优化（`stop_period=10`、`stop_atr_multiple=2.0` 等）
- **累计亏损38%问题**：回测日志显示后期累计亏损达 38.03%，超过 15% 阈值后永久冻结——说明 P0-1（累计亏损复位）已在修复中正确处理，但 `max_cumulative_loss_pct=15%` 的阈值可能过小，可考虑在网格搜索中调大
- **最终净值 ¥218,761 vs 初始 ¥200,000**：6.5年+9.38% 年化约 1.4%，仍有提升空间（需要配合 S6 参数优化）
